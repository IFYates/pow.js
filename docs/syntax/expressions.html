Internal values

v1.x

The original design of only handled very basic expression parsing, so using a non-standard character to differentiate internal values from normal data was useful.
This does mean that some expressions are not possible, such as *root.attr1 == *data.attr2.

*root

v2.x

In order to make better use of the full expression support in , the design was revised to use only standard characters, and expose the internal values as normal properties.

$root

pow.safe

Full expression support requires a fully trusted browser environment. In situations where this isn't possible, can be used to replace dynamic expression parsing with some simpler logic.
