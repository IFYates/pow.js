Syntax / Bindings

exposes two functions for binding document elements:

  • bind(element) Returns a binding object for the given element hierarchy.
  • apply(element, data) Applies the data to the given element hierarchy and returns the binding object.

The binding object exposes the following functions:

  • apply(data) Applies data to the bound element hierarchy.
  • refresh() Returns a binding object for the given element hierarchy.

The data is applied to the entire element hierarchy; with additional binding only being applied to elements that have a pow attribute.

The possible binding attributes are:

  • stop
  • if ifnot else else-if else-ifnot
  • template
  • item
  • data
  • array
  • template

This is also the order that they will be evaluated in; so, the context of an if will not be altered by any {{ version(2.0) ? 'data' : 'item' }}/array expression.

{{ version(2.0) ? 'data' : 'item' }}="expression"

Sets the data context for the binding of the hierarchy from this element. 

    <!-- For { child: { text: 'Hello, world!' } } -->
    <div pow {{ version(2.0) ? 'data' : 'item' }}="child">
        {{ text }}
    </div>

    <!-- Result -->
    <div>
        Hello, world!
    </div>

Because the attribute takes an expression, the value does not have to be in the data context. 

    <div pow {{ version(2.0) ? 'data' : 'item' }}="{ text: 'Hello, world!' }">
        {{ text }}
    </div>

    <!-- Result -->
    <div>
        Hello, world!
    </div>

If the expresson evaluates to null, the element will not be rendered.

stop

Prevents any further bindings from being applied to the element hierarchy. 

    <div pow stop>
        {{ text }}
    </div>

    <!-- Result -->
    <div pow stop>
        {{ text }}
    </div>

If an explicit call to apply() is made for this element, the stop attribute will be ignored and the hierarchy will be processed.

This enables you to apply a different binding to part of an element hierarchy that has already been bound.
