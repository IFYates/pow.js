Breaking changes

v3.0

  • else-if and else-ifnot attributes have been removed, as they were unnecessary.
    Combine else and if or ifnot on the same element to achieve the same result.
  • The template binding no longer parses its content if the id cannot be matched.
    Use a <template pow else> sibling to provide this behaviour.

v2.0

  • item attribute has been renamed to data, to better match its behaviour.
  • Binding attributes are now parsed in the order they appear in the HTML, instead of based on a fixed order.
  • Bound functions now receive the entire context object as an argument, instead of the $data and $root items separately.
    Previous behaviour can be restored by using the function({ $data, $root }) signature.
  • The template binding now replaces the content of the bound element rather than the entire element.
    This allows setting other attributes on the bound element, as well as an enclosing tag.
    Use a <template> element to achieve the previous behaviour.
